# 척도검사 웹앱 구글 시트 연동

정적 웹앱 `scale-screening-web-app`의 검사 기록을 구글 시트에 저장하는 운영 절차입니다.

## 구성

- 프런트엔드: 브라우저에서 검사 결과/저장 기록을 로컬 Node 서버로 전송
- 서버 연동 방식
  - 방법 A: Google Sheets API 직접 호출
  - 방법 B: Apps Script Web App 프록시
- 시트 구조
  - `척도검사기록`: 검사 1건당 1행
  - `척도문항응답`: 문항과 하위문항 응답 상세
  - `척도마스터`: 척도 기본 정보
  - `척도문항마스터`: 응답자 정보/문항/하위문항 구조
  - `척도선택지마스터`: 각 항목 선택지 구조

## 중요

- `Google Sheets API 키만`으로는 새 시트 생성이나 시트 쓰기가 되지 않습니다.
- 직접 연동은 OAuth 2.0 또는 서비스 계정이 필요합니다.

## 1. Google Sheets API 직접 연동

1. Google Cloud에서 `Google Sheets API`와 `Google Drive API`를 켭니다.
2. 서비스 계정을 만들고 JSON 키를 발급합니다.
3. 서버 환경변수에 아래 값을 설정합니다.
   - `MH_GOOGLE_SERVICE_ACCOUNT_JSON`
   - 또는 `MH_GOOGLE_SERVICE_ACCOUNT_EMAIL`
   - 또는 `MH_GOOGLE_SERVICE_ACCOUNT_PRIVATE_KEY`
4. 자동 생성 모드를 쓰려면 아래 둘 중 하나를 추가합니다.
   - `MH_GOOGLE_SYNC_SPREADSHEET_ID`: 기존 대상 시트 ID
   - `MH_GOOGLE_SYNC_SHARE_WITH_EMAIL`: 새로 만든 시트를 공유할 본인 계정 이메일
5. 선택 값
   - `MH_GOOGLE_SYNC_SPREADSHEET_TITLE`: 자동 생성 시 제목
6. 서버를 다시 시작합니다.

이 모드에서는 첫 전송 시 서버가 아래 5개 시트를 자동으로 준비합니다.

- `척도검사기록`
- `척도문항응답`
- `척도마스터`
- `척도문항마스터`
- `척도선택지마스터`

## 2. Apps Script 반영

1. 대상 구글 스프레드시트에서 `확장 프로그램 > Apps Script`를 엽니다.
2. `google-apps-script` 폴더의 `.gs` 파일을 반영합니다.
3. 저장 후 스프레드시트를 새로 엽니다.

## 3. 시트와 토큰 준비

1. 상단 메뉴 `데이터 자동화 > 척도검사 시트 준비`를 실행합니다.
2. `데이터 자동화 > 현재 시트를 척도검사 DB로 지정`을 실행합니다.
3. 필요하면 `데이터 자동화 > 척도검사 토큰 저장`을 실행합니다.
4. 토큰을 쓸 경우 웹앱과 동일하게 사용할 토큰 문자열을 저장합니다.
5. 필요하면 `데이터 자동화 > 척도검사 연동 상태 보기`로 준비 상태를 확인합니다.

## 4. Web App 배포

1. Apps Script 우상단 `배포 > 새 배포`를 클릭합니다.
2. 유형을 `웹 앱`으로 선택합니다.
3. 아래처럼 설정합니다.
   - 실행 사용자: 본인
   - 액세스 권한: 링크가 있는 모든 사용자
4. 배포 후 발급된 Web App URL을 복사합니다.

## 5. 웹앱 연결

1. `scale-screening-web-app`을 실행합니다.
2. 서버에 direct API 또는 Apps Script 환경변수를 설정합니다.
3. 먼저 `척도 마스터 전송`을 눌러 8개 척도 원본 구조를 시트에 적재할 수 있습니다.
4. 현재 결과만 보내려면 `현재 결과 전송`을 누릅니다.
5. 저장해 둔 최근 기록 전체를 보내려면 `저장 기록 전체 전송`을 누릅니다.
6. 결과 전송 요청에도 척도 마스터가 함께 포함되므로, 척도 시트는 계속 최신 상태로 갱신됩니다.

## 저장 방식

- 같은 `record_id`가 다시 전송되면 `척도검사기록`과 `척도문항응답`에서 기존 행을 갱신합니다.
- 같은 `questionnaire_id`, `field_key`, `option_key`가 다시 전송되면 척도 마스터 시트도 기존 행을 갱신합니다.
- 각 검사 기록은 요약 컬럼과 함께 전체 JSON 원본도 `record_json` 컬럼에 저장됩니다.
- 브라우저 특성상 전송은 `fire-and-forget` 방식으로 이뤄지므로, 운영 시에는 전송 후 시트 반영 여부를 한 번 확인하는 흐름을 권장합니다.
- Apps Script 모드에서 토큰이 없으면 설정은 단순하지만 Web App URL을 아는 사람이 기록을 보낼 수 있으니 내부 운영용으로만 권장합니다.
